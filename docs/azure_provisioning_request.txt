1. WHAT THIS PROJECT DOES
================================================================================

The AutoStore Support Agent monitors a shared Outlook inbox, reads
incoming customer emails about AutoStore API integration, searches our
published documentation for answers, and generates draft replies for
human review. It does NOT auto-send — a team member always reviews
and sends.

Flow:
    Shared Inbox → Agent reads email
                 → Classifies the topic (Azure OpenAI)
                 → Searches AutoStore docs (ReadMe MCP server — already live)
                 → Generates a draft reply (Azure OpenAI)
                 → Saves draft in the shared inbox
                 → Human reviews and sends

4. RESOURCE 2 — AZURE ENTRA ID APP REGISTRATION
================================================================================

Purpose:    Read emails and create drafts in a shared Outlook mailbox
            using Microsoft Graph API (application-level, no user login)

Target mailbox: integration.support@autostoresystem.com
    (or whichever shared mailbox the team uses)

Steps:
    1. Register a new app in Azure Entra ID (Azure AD)
       - Name: "AutoStore Support Agent"
       - Supported account type: Single tenant

    2. Add API permissions (APPLICATION type, not delegated):
       - Microsoft Graph > Mail.ReadWrite    (read emails, create drafts)
       - Microsoft Graph > Mail.Send         (optional, for future auto-send)

    3. Grant admin consent for the permissions

    4. Create a client secret
       - Note the expiration date — set a calendar reminder to rotate

    5. (Recommended) Scope access to the shared mailbox only using an
       Application Access Policy so the app cannot read other mailboxes:

       New-ApplicationAccessPolicy `
           -AppId "<client_id>" `
           -PolicyScopeGroupId "integration.support@autostoresystem.com" `
           -AccessRight RestrictAccess `
           -Description "Limit Support Agent to shared mailbox only"

What we need returned:
    - OUTLOOK_CLIENT_ID       (Application/client ID)
    - OUTLOOK_CLIENT_SECRET   (Client secret value)
    - OUTLOOK_TENANT_ID       (Directory/tenant ID)
    - SHARED_MAILBOX          (e.g., integration.support@autostoresystem.com)

5. RESOURCE 3 — AZURE HOSTING
================================================================================

Purpose:    Run the Python agent as a long-running service

Option A — Azure Container Apps (recommended):
    - Single container running the Python polling loop
    - Min replicas: 1 (always on during business hours)
    - Can scale to 0 outside business hours to save cost
    - Environment variables loaded from Key Vault

Option B — Azure Functions (timer trigger):
    - Timer-triggered function running every 5 minutes
    - Consumption plan (pay per execution) — cheapest option
    - Cold start may add 10-15s latency per poll cycle

Requirements for either option:
    - Python 3.12+ runtime
    - Outbound internet access to:
        * https://interface.autostoresystem.com (ReadMe MCP server)
        * https://*.openai.azure.com (Azure OpenAI)
        * https://graph.microsoft.com (Microsoft Graph / Outlook)
        * https://login.microsoftonline.com (Entra ID auth)
    - No inbound ports needed (agent polls outbound only)

Resource sizing (minimal):
    - CPU: 0.25 vCPU
    - Memory: 0.5 GB
    - Storage: minimal (no local storage needed)
